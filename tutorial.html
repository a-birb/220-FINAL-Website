Installing Active Directory Certificate Authority & Installing a Web Server Certificate


Below are a series of videos that demonstrate the process of installing the Active Directory Certificate Authority into a domain environment. Before watching, ensure that you are in a domain-enabled environment with administrative access. You should only perform these tasks with permission from the administrator, and this method should only be used on internal webservers- not for websites intended for use on the internet!


In this video, I demonstrate:

  • Installing and configuring Active Directory Certificate Services
  • The Certificate and Certificate Authority snap-ins in the Microsoft Management Console
  • Creating an advanced Certificate Request using the Management Console
  • Signing a Certificate Request using the command line on the CA server

Note that there are always multiple methods to creating and signing certificates. The Custom Certificate Request option in the MMC was used in order to correctly configure Subject Alternative Names for the website's certificate; without this additional configuration, modern web browsers WILL reject the certificate.

In this video I demonstrate:

  • Installing a signed certificate using the IIS Management GUI
  • Adding custom HTML content to an IIS website
  • Applying an HTTPS binding and hostname to an IIS website
  • Browsing using HTTPS on the local server

In this video I demonstrate:

  • Configuring a DNS record for the website's hostname
  • Distributing the Root CA Certificate using Group Policy
  • How to verify that HTTPS is in use on a modern web browser
